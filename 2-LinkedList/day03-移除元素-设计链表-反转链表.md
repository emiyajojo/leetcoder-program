# 移除元素
虚拟头节点，单指针，while循环中单指针pointer的**移动和删除操作是互斥**的，**不能**在一次循环中**既移动又删除**，**否则会变空**

# 设计链表
首先，链表在初始化的时候 **需要虚拟头节点以及链表长度**
`addAtHead()`是最简单的,就直接在虚拟头节点后面添加一个节点即可，记得初始化的时候要给next赋值
`addAtTail()`也是不难，只需要遍历到最后一个节点并新添一个新节点就行
`get(), addAtIndex(), deleteAtIndex()` 的不同的边界条件、移动指针初始化以及for终止条件：
- **共同点**: 是他们都使用**for循环依靠index**来遍历, 而不是while依靠节点非空为终止条件遍历
- `get()`由于是取index指定的链表元素,因此while需要遍历到index所指向的元素, 另外两个则需要遍历到 **index之前的元素**, 并且为了对应 index==0的情况, pointer 初始化是 虚拟.next, 边界条件为index小于0或index不大于最后一个链表元素下标
- `addAtIndex()`, 遍历到index之前, pointer初始化为虚拟节点, 正好对应 **index之前** 的思想
- `deleteAtIndex()`, 依然遍历到index之前, 虽然是删除 index 指向的节点, 但是操作上, 需要的是index前面的节点, 边界条件除了小于0以外则是 index **不大于最后的下标+1**, 若刚好为最后下标+1, 执行尾部添加
